# 1차시 과제: 나만의 파이프라인 지도 그리기

이번 과제는 구현 과제가 아닙니다. 1차시에서 본 전체 지도를 본인 언어로 다시 그려보고, 각 기술이 왜 그 자리에 있는지 설명해보는 과제입니다.

잘 그리는 것이 목적이 아니라 전체 흐름을 스스로 설명해보는 것이 목적입니다. 헷갈리는 부분은 물음표로 남겨도 괜찮습니다.

## 제출 내용

### 1. 전체 아키텍처 그림

아래 흐름을 포함해 본인 방식으로 그립니다.

```text
Commerce Events
  -> Kafka
  -> Flink
  -> Paimon
  -> Spark / Airflow
  -> Iceberg
  -> StarRocks
  -> BI
```

각 단계 옆에 "이 기술이 맡는 역할"을 한 줄씩 적어주세요.

가능한 형식:

- 손그림 사진
- Excalidraw
- draw.io
- PPT
- Mermaid
- Notion
- 이미지 파일

### 2. Realtime OLAP vs Batch Lakehouse BI 차이

슬라이드 문장을 그대로 복사하지 말고, 본인 말로 한두 줄씩 정리합니다.

- Realtime OLAP은 어떤 질문에 답하는 경로인가?
- Batch Lakehouse BI는 어떤 질문에 답하는 경로인가?

### 3. 운영에서 생길 것 같은 어려움 예측

아직 직접 만들기 전이므로 정답일 필요는 없습니다. 지금의 직감으로 하나를 고릅니다.

- 가장 먼저 깨질 것 같은 단계는 어디인가?
- 어떤 상황에서 문제가 생길 것 같은가?
- 문제가 생기면 어디부터 확인할 것 같은가?
- 그렇게 생각한 이유는 무엇인가?

예시:

- Kafka 적체
- Flink 중단 시 Paimon 적재 상태
- 작은 파일 누적
- Batch 결과와 Realtime 결과 차이
- StarRocks 역할 혼동

여기 적은 예측은 6차시 장애 대응 시간에 실제 결과와 비교합니다.

### 4. 본인 실습 환경 정보

아래 정보를 적어주세요.

- OS
- CPU
- RAM
- Docker Desktop 설치 및 실행 여부
- Git 설치 여부

가능하면 아래 명령 실행 결과도 함께 첨부합니다.

```bash
git --version
docker --version
docker ps
```
