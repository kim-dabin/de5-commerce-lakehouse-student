개념 정리Lakehouse

Lakehouse는 파일 더미를 운영 가능한 테이블로 바꾸는 방식입니다

Data Lake

원천 데이터를 많이 저장하기 좋지만, 파일만 쌓이면 관리가 어려워집니다.

DWH

정제된 데이터를 빠르게 분석하기 좋지만, 원천 보관과 재처리 유연성은 제한될 수 있습니다.

Lakehouse

Data Lake 위에 테이블 포맷을 올려 snapshot, schema, partition, 복구 기준을 관리합니다.

파일 더미에 주소, 규칙, 이력, 복구 기준을 붙인다.
발표 메모
  • Parquet 파일만 있으면 최신 파일/스키마/되돌리기 기준을 관리하기 어렵다고 설명한다.
  • Paimon과 Iceberg는 이 테이블 포맷 영역에 있다고 연결한다.

사전 가이드 핵심Paimon vs Iceberg

둘 다 테이블 포맷이지만, 이번 프로젝트에서는 역할을 나눕니다

Realtime Bronze

Paimon

  • Kafka와 Flink 뒤에서 실시간 데이터를 받아둔다.
  • 변경이 계속 들어오는 L0 Bronze에 둔다.
  • checkpoint, changelog, compaction을 이후에 본다.
Batch Canonical

Iceberg

  • Spark 배치가 만든 기준 테이블을 저장한다.
  • 정제본과 집계본을 snapshot 기반으로 관리한다.
  • time travel, partition, rollback을 이후에 본다.

사전 가이드는 공유되는 대로 1~3장과 7장 비교표만 먼저 읽으면 충분합니다.

발표 메모
  • “Paimon은 실시간 장부, Iceberg는 결산 장부” 문장을 사용한다.
  • 나머지 상세는 4차시와 5차시에서 다시 돌아온다고 말한다.
