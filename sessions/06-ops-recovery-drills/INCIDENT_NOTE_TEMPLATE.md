# 6차시 인시던트 노트 템플릿

아래 형식으로 라운드 1개 이상을 정리합니다.

## 기본 정보

- 이름:
- 실행 환경:
- 선택한 라운드: R1 / R2 / R3 / R4 / R5 / R6

## 인시던트 요약

```text
장애:
내가 기대한 정상 상태:
실제로 관찰한 증상:
```

## 증거

```text
Kafka 증거:
Flink 증거:
Paimon 증거:
Iceberg/StarRocks/BI 증거:
```

해당 없는 계층은 `해당 없음`이라고 적어도 됩니다.

## 원인 가설

```text
처음 의심한 곳:
그렇게 생각한 이유:
나중에 배제한 원인:
최종 원인:
```

## 복구

```text
실행한 복구 명령:
복구 후 확인한 증거:
남은 리스크:
```

## 포트폴리오 문장

아래 문장을 본인 말로 완성합니다.

```text
이 장애에서 배운 점은 ______ 입니다.
왜냐하면 ______ 계층은 정상이어도 ______ 계층은 실패할 수 있기 때문입니다.
그래서 운영에서는 ______ 를 확인한 뒤 ______ 를 실행해야 합니다.
```

## 최종 발표 슬라이드 초안

아래 구조를 그대로 발표 자료 1-2장으로 옮겨도 됩니다.

```text
슬라이드 제목:

1. 장애/운영 리스크:
2. 처음 보인 증상:
3. 계층별 증거:
   - Kafka:
   - Flink:
   - Paimon:
   - Iceberg/StarRocks/BI/Airflow:
4. 복구 명령:
5. 복구 후 확인한 결과:
6. 재발 방지 또는 개선 아이디어:
7. 최종 발표에서 말할 한 문장:
```
