# 8차시 과제 - 최종 포트폴리오 발표 제출

그동안 차시별로 쌓은 과제(증거, 인시던트 노트, 품질/serving 해석)를 하나의 포트폴리오 발표로 묶어 제출합니다.

## 제출물

1. **발표 슬라이드** (7장 권장, `PRESENTATION_TEMPLATE.md` 기반) — PDF / Keynote / PPT / Google Slides / Notion 중 편한 형식.
2. **프로젝트 README** (선택이지만 권장) — 실행 방법 + 아키텍처 + 내 질문 + known gap 요약.

형식보다 "질문 → 구조 → 증거 → 해석 → 개선" 흐름이 살아 있는지가 중요합니다.

## 필수 내용 (슬라이드에 모두 포함)

1. **내 질문 1개** — 공통 데이터에서 내가 집중한 비즈니스 질문.
2. **아키텍처** — 공통 파이프라인 흐름 + 내 질문이 답해지는 지점.
3. **데이터 모델/grain** — 내가 읽은 지표가 어느 mart·어느 grain에서 나왔는지.
4. **BI 결과 해석** — 숫자를 그대로 읽지 말고 질문에 연결.
5. **계층별 검증 증거** — `EVIDENCE_CARD.md`로 직접 확인한 계층만 (Kafka/Flink/Paimon/Iceberg/품질/StarRocks/Airflow 중).
6. **리스크 + 개선 로드맵** — known gap 1~2개와 우선순위 있는 개선안.
7. **한 문장 마무리** — 이 프로젝트에서 내가 가장 중요하게 본 것.

## 평가 기준 (루브릭)

정답을 맞히는 과제가 아닙니다. 아래 관점으로 봅니다.

| 항목 | 약함 | 좋음 |
|---|---|---|
| 질문 | 질문 없이 기능 나열 | 명확한 비즈니스 질문에서 출발 |
| 지표 해석 | 대시보드 캡처만 | 어느 mart·grain에서 나온 숫자인지 설명 |
| 검증 증거 | "돌아갔습니다" | 계층별 증거로 "어디까지 정상"을 입증 |
| 리스크 | 갭을 숨김 | 갭을 인정하고 우선순위 있는 개선으로 표현 |
| 메시지 | 파이프라인 순서 암송 | 내 관점이 한 문장으로 남음 |

특히 **"완벽한 시스템"보다 "완성한 것과 남은 갭을 구분하고 우선순위를 말하는 것"**을 높게 봅니다.

## 차시 과제를 재료로

- 6차시 인시던트 노트 → 슬라이드 6(검증 증거) / 슬라이드 7(리스크).
- 7차시 품질 규칙 + serving 해석 → 슬라이드 5(해석) / 슬라이드 6(품질 게이트 증거).

새 프로젝트를 급히 시작하지 말고, 쌓아온 것을 묶으세요.
